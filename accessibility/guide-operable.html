원칙 2. 운용의 용이성 (Operable)

모든 사용자가 쉽게 운용할 수 있어야 합니다

2.1 입력장치 접근성

모든 기능은 키보드만으로도 사용할 수 있어야 합니다.

2.1.1 키보드 사용 보장

마우스 이벤트 핸들러(onclick, onmouseover 등)와 함께 키보드 이벤트 핸들러를 제공해야 합니다.

❌ 잘못된 사례 - 마우스로만 동작

<div onclick="showMenu()">메뉴 열기</div>

✅ 올바른 사례 - 키보드로도 접근 가능

<button onclick="showMenu()" onkeypress="showMenu()">메뉴 열기</button>

2.1.2 초점 이동

키보드 포커스는 논리적인 순서로 이동해야 하며, 포커스가 보이도록 해야 합니다.

✅ 올바른 사례 - 포커스 표시

button:focus { outline: 2px solid #059669; outline-offset: 2px; }

2.1.3 조작 가능

사용자 입력 및 컨트롤은 조작 가능하도록 제공되어야 합니다.

2.2 충분한 시간 제공

콘텐츠를 읽고 사용하는데 충분한 시간을 제공해야 합니다.

2.2.1 응답시간 조절

시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있는 수단을 제공해야 합니다.

✅ 올바른 사례 - 세션 만료 전 경고 및 연장 옵션

<div role="alert"> <p>세션이 1분 후 만료됩니다.</p> <button>세션 연장하기</button> </div>

2.2.2 정지 기능 제공

자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있는 수단을 제공해야 합니다.

✅ 올바른 사례 - 자동 슬라이드에 정지/재생 버튼

<div class="carousel"> <button aria-label="슬라이드 정지">정지</button> <button aria-label="슬라이드 재생">재생</button> </div>

2.3 광과민성 발작 예방

광과민성 발작을 일으킬 수 있는 콘텐츠를 제공하지 않아야 합니다.

2.3.1 깜빡임과 번쩍임 사용 제한

초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 합니다.

❌ 잘못된 사례 - 빠르게 깜빡이는 애니메이션

@keyframes blink { 0%, 50% { opacity: 1; } 25%, 75% { opacity: 0; } } .flash { animation: blink 0.1s infinite; }

2.4 쉬운 네비게이션

콘텐츠를 쉽게 찾고 탐색할 수 있어야 합니다.

2.4.1 반복 영역 건너뛰기

콘텐츠의 반복되는 영역은 건너뛸 수 있어야 합니다.

✅ 올바른 사례 - Skip Navigation 제공

<a href="#main-content" class="sr-only">본문 바로가기</a> <header>...</header> <main id="main-content">...</main>

2.4.2 제목 제공

페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 합니다.

✅ 올바른 사례 - 명확한 페이지 제목

<title>웹 접근성 컨설팅 - 웹소울랩</title>

2.4.3 적절한 링크 텍스트

링크 텍스트는 링크의 목적을 이해할 수 있도록 제공해야 합니다.

❌ 잘못된 사례

<a href="guide.pdf">여기를 클릭하세요</a>

✅ 올바른 사례
