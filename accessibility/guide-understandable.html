원칙 3. 이해의 용이성 (Understandable)

콘텐츠는 이해할 수 있어야 합니다

3.1 가독성

텍스트 콘텐츠는 읽고 이해할 수 있어야 합니다.

3.1.1 기본 언어 표시

주로 사용하는 언어를 명시해야 합니다.

✅ 올바른 사례 - html 태그에 lang 속성 제공

<html lang="ko">

3.1.2 외국어 표기

본문에서 사용된 외국어는 언어를 명시해야 합니다.

✅ 올바른 사례

<p>웹 접근성을 의미하는 <span lang="en">Web Accessibility</span></p>

3.2 예측 가능성

웹 페이지의 기능은 예측 가능하도록 구성되어야 합니다.

3.2.1 사용자 요구에 따른 실행

사용자가 의도하지 않은 기능이 실행되지 않도록 해야 합니다.

❌ 잘못된 사례 - 포커스만으로 새 창 열림

<a href="popup.html" onfocus="window.open(this.href)">링크</a>

✅ 올바른 사례 - 클릭 시 새 창 열림 (사전 안내)

<a href="popup.html" target="_blank">링크 (새 창으로 열림)</a>

3.2.2 콘텐츠의 선형 구조

콘텐츠는 논리적인 순서로 제공되어야 합니다.

✅ 올바른 사례 - 논리적 heading 구조

<h1>웹 접근성 가이드</h1> <h2>1. 인식의 용이성</h2> <h3>1.1 대체 텍스트</h3> <h3>1.2 멀티미디어</h3> <h2>2. 운용의 용이성</h2>

3.2.3 일관성 있는 네비게이션

웹 페이지 간 일관된 메뉴 위치와 순서를 유지해야 합니다.

3.3 입력 도움

사용자 입력 시 오류를 방지하거나 정정할 수 있도록 해야 합니다.

3.3.1 오류 정정

입력 오류 발생 시 오류를 정정할 수 있는 방법을 제공해야 합니다.

✅ 올바른 사례 - 오류 메시지와 정정 방법 제공

<label for="email">이메일 *</label> <input type="email" id="email" aria-describedby="email-error"> <p id="email-error" class="error">유효한 이메일 형식이 아닙니다. 예: user@example.com</p>

3.3.2 레이블 제공

모든 입력 서식에는 대응하는 레이블을 제공해야 합니다.

✅ 올바른 사례

<label for="username">사용자명</label> <input type="text" id="username" name="username">

3.3.3 오류 예방

법적, 금융 거래 등 중요한 입력은 제출 전 확인, 수정, 취소 기회를 제공해야 합니다.

✅ 올바른 사례 - 최종 확인 단계 제공

<div class="confirmation"> <h2>결제 정보 확인</h2> <p>상품명: 웹 접근성 컨설팅</p> <p>금액: 5,000,000원</p> <button>수정하기</button> <button>결제하기</button> </div>
