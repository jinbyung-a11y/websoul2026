원칙 4. 견고성 (Robust)

다양한 기술로 접근할 수 있어야 합니다

4.1 문법 준수

마크업 언어의 문법을 준수해야 합니다.

4.1.1 마크업 오류 방지

HTML, CSS 등의 마크업 언어는 해당 문법을 준수해야 합니다.

❌ 잘못된 사례 - 태그 닫기 오류

<div> <p>콘텐츠</div> </p>

✅ 올바른 사례 - 올바른 태그 닫기

<div> <p>콘텐츠</p> </div>

4.1.2 중복 ID 사용 금지

같은 페이지 내에서 동일한 ID 값을 중복 사용하지 않아야 합니다.

❌ 잘못된 사례 - ID 중복

<div id="content">...</div> <section id="content">...</section>

✅ 올바른 사례 - 고유한 ID 사용

<div id="main-content">...</div> <section id="sub-content">...</section>

4.1.3 표준 문법 준수

W3C HTML5 표준 문법을 준수해야 합니다.

✅ 올바른 사례 - HTML5 DOCTYPE 선언

<!DOCTYPE html> <html lang="ko"> <head> <meta charset="UTF-8"> <title>페이지 제목</title> </head> <body> ... </body> </html>

4.2 웹 애플리케이션 접근성

콘텐츠는 다양한 사용자 환경에서 접근 가능해야 합니다.

4.2.1 웹 애플리케이션 접근성

부가 기능 없이 서비스 이용이 가능하도록 구성하거나, 대체 수단을 제공해야 합니다.

❌ 잘못된 사례 - 플러그인 필수

<embed src="content.swf" type="application/x-shockwave-flash">

* Flash 등 플러그인이 없으면 콘텐츠 이용 불가

✅ 올바른 사례 - 표준 기술 사용

<video controls> <source src="video.mp4" type="video/mp4"> <p>브라우저가 비디오 태그를 지원하지 않습니다.</p> </video>

4.2.2 보조 기술 호환성

스크린리더 등 보조 기술과 호환 가능하도록 구현해야 합니다.

✅ 올바른 사례 - ARIA 역할 및 상태 제공

<button aria-expanded="false" aria-controls="menu"> 메뉴 열기 </button> <div id="menu" aria-hidden="true"> <ul>...</ul> </div>

4.2.3 프로그래밍적으로 결정 가능

사용자 인터페이스 컴포넌트는 프로그래밍적으로 제어 가능해야 합니다.

✅ 올바른 사례 - 적절한 의미의 요소 사용

<nav aria-label="주 메뉴"> <ul> <li><a href="/">홈</a></li> <li><a href="/about">회사소개</a></li> </ul> </nav>

웹 표준 검증

W3C Markup Validation Service를 통해 HTML 문법을 검증할 수 있습니다.

검증 도구:

  • W3C Markup Validator: validator.w3.org
  • W3C CSS Validator: jigsaw.w3.org/css-validator
